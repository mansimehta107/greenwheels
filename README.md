## 🗂️ Project Structure

```
greenwheels/
├── index.html                          → Root redirect → blog/
├── README.md
└── blog/
    ├── index.html                      → Blog hub / listing page
    ├── css/blog-styles.css             → Shared stylesheet (Green Wheels branded)
    ├── images/                         → Hero images (.webp)
    ├── christmas-escooter-gift/
    │   ├── index.html                  → Full HTML article
    │   └── content.md                  → Markdown source + YAML front-matter
    ├── valentines-day-escooter-gift/
    ├── fathers-day-escooter-gift/
    ├── mothers-day-escooter-gift/
    ├── birthday-escooter-gift/
    ├── graduation-escooter-gift/
    ├── back-to-school-escooter-gift/
    ├── easter-escooter-gift/
    ├── st-patricks-day-escooter-gift/
    └── anniversary-escooter-gift/
```

> Each article folder contains an **`index.html`** (fully styled, self-contained) and a **`content.md`** (Markdown + YAML front-matter for CMS integration).

## ⚡ Tech Stack

| Layer | Technology |
|---|---|
| **Markup** | Semantic HTML5 |
| **Styling** | CSS3 with Custom Properties |
| **Fonts** | Google Fonts (Montserrat + Open Sans) |
| **Images** | WebP hero images |
| **Design** | Mobile-first responsive |
| **Hosting** | GitHub Pages (auto-deploy from `main`) |

> No build step. No frameworks. No dependencies. Just open and go.

## 🚀 Quick Start

```bash
# Clone the repo
git clone https://github.com/mansimehta107/greenwheels.git
cd greenwheels

# Open directly in browser
open blog/index.html

# Or start a local server
python3 -m http.server 8000
# Visit http://localhost:8000/blog/
```

## 🔍 SEO Features

Every article ships with:

- [x] Unique `<title>` and `<meta description>`
- [x] Targeted keyword meta tags
- [x] Open Graph tags (`og:title`, `og:description`, `og:image`)
- [x] Semantic headings (H1 → H2 → H3)
- [x] Alt text on all images
- [x] Internal links to product pages
- [x] Mobile-responsive layout
- [x] Fast load times (no JS frameworks)

---

## 📦 Deployment

Deployed automatically via **GitHub Pages** from the `main` branch. Every push to `main` triggers a rebuild — no CI/CD configuration needed.
